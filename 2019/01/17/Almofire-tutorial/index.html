Alamofire教程:入门

通过使用 Imagga API 上传和分析用户照片，带你进入Alamofire，这是一个在iOS上支持数千个应用程序的网络库。

Posted by cynine on 2019-01-17

作者: Ron Kliffer 原文链接

  • 更新说明: Ron Kliffer 将本教程更新为 Xcode 9.3、iOS 11.3、Swift 4.1 和Alamofire 4.7.0。最初的教程是由 Aaron Douglas 编写的。

Alamofire 是一个基于 Swift 的用于 iOS 和 macOS 的 HTTP 网络库。它在苹果的基础网络栈上提供了一个优雅的界面，简化了很多常见的网络任务。

Alamofire 提供了可链接的请求/响应方法、JSON 参数和响应序列化、身份验证以及许多其他特性。

在这个 Alamofire 教程中，你将使用 Alamofire 执行基本的网络任务，如上传文件和从第三方请求 API 数据。

Alamofire 的优雅之处在于，它是用 Swift 从头开始编写的，没有从 Objective-C 对应的 AFNetworking 继承任何东西。

你应该对 HTTP 网络有一个概念性的理解，并对 Apple 的网络类(如: URLSession)有一定的了解。

虽然 Alamofire 确实掩盖了一些实现细节，但如果你需要对网络请求进行故障排除，最好了解一些背景知识。

The Imagga API

Imagga 是一个图像识别平台即服务，它为开发人员和企业提供图像标记api来构建可伸缩的、图像密集型的云应用程序。你可以在这里试用他们的自动标签服务。

对于本 Alamofire 教程，你需要使用 Imagga 创建一个免费的开发人员帐户。Imagga 在每个 HTTP 请求中都需要一个授权头，这样只有拥有帐户的人才可以使用他们的服务。访问 https://imagga.com/auth/signup/hacker 并填写表单。创建帐户后，查看仪表板:

在授权部分下面列出的是你稍后将使用的秘密令牌。你需要将此信息作为头包含在每个HTTP请求中。

注意: 确保您复制了整个秘密令牌，确保向右滚动并验证您复制了所有内容。

您将使用 Imagga 的内容端点来上传照片，为图像识别标记端点，为颜色识别标记端点。您可以在 http://docs.imagga.com 上阅读关于 Imagga API 的所有信息。
